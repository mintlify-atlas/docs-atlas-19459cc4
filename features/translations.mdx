Rosetta is Launchpad's translation platform. It provides a web-based interface for translators, supports importing and exporting industry-standard translation file formats, and integrates with the upstream projects and Ubuntu distribution packages it translates.

## Translation permission models

Projects and distributions can control how open their translation process is (`TranslationPermission`):

| Permission | Behaviour |
|---|---|
| `Open` | Any logged-in user can add or edit translations in any language without review. |
| `Structured` | Designated translators exist for some languages; in those languages, non-designated users can only make suggestions. In languages without a designated translator, anyone can edit directly. |
| `Restricted` | Only designated translators can edit translations. Others can submit suggestions, which must be reviewed before acceptance. |
| `Closed` | Only designated translators can add or edit translations. Non-designated users cannot even submit suggestions. |

## Supported file formats

Rosetta reads and writes several translation file formats (`TranslationFileFormat`):

| Format | Description |
|---|---|
| `PO` | Gettext standard text format (`.po` files). |
| `MO` | Gettext compiled binary format (`.mo` files). |
| `XPI` | Mozilla XPI extension format used by Firefox and Thunderbird. |
| `KDE PO` | Legacy KDE PO format that embeds context and plural forms in messages instead of using standard gettext features. |
| `XPI PO` | Variant of gettext that always uses English strings as `msgid` values, useful when exporting XPI translations to the gettext world. |
| `PO (changed only)` | Partial PO file containing only messages changed compared to the imported version. |

<Note>
POT files (template files) are the source of truth. Launchpad stores the template and generates per-language PO files from it.
</Note>

## Import queue

When you upload a translation file, it enters the Rosetta import queue. Each entry has a status (`RosettaImportStatus`):

| Status | Meaning |
|---|---|
| `Approved` | Approved by a Rosetta expert or automatically; waiting to be imported. |
| `Imported` | Successfully imported into the database. |
| `Needs Review` | Requires manual review by a Rosetta expert before import. |
| `Needs Information` | Reviewer needs more information before deciding. |
| `Blocked` | Explicitly blocked from import by a Rosetta expert. |
| `Failed` | Import attempt failed (e.g., malformed file). |
| `Deleted` | Entry removed from the queue before import. |

## Importing translations

<Steps>
  <Step title="Navigate to the translation templates">
    Open your project or package in Launchpad and click **Translations**, then select the series you want to work with.
  </Step>
  <Step title="Upload a PO or POT file">
    Click **Import a file** and upload your `.pot` (template) or `.po` (language) file. You can also import a tarball containing multiple files.
  </Step>
  <Step title="Wait for queue processing">
    The file enters the import queue with status `Needs Review` or is automatically approved. You can monitor the queue at `translations.launchpad.net/import`.
  </Step>
  <Step title="Automatic approval">
    Files that match an existing template and have a known language code are usually approved automatically. Files for new templates require manual review.
  </Step>
</Steps>

## Exporting translations

You can export translations in PO or MO format at any time:

- **Single language**: Click **Download** on a language page to get that language's PO file.
- **Full export**: Request a tarball of all translations for a template from the template page. Launchpad generates the tarball asynchronously and emails you a download link.
- **Language packs**: Ubuntu releases include language packs generated by Launchpad. Full packs (`LanguagePackType.FULL`) contain all translations; delta packs (`LanguagePackType.DELTA`) contain only changes since the last full export.

## Translation teams and coordinators

Each language in a project can have a **translation team** — a Launchpad team whose members are the designated translators for that language.

Translation groups (`TranslationGroup`) are collections of per-language translator teams managed by a group owner. A project can adopt an existing translation group rather than managing its own.

Team roles:

- **Team members** — Can translate directly (in `Restricted` or `Closed` mode) or review suggestions.
- **Coordinators** — Manage team membership and approve or reject suggestions.

## Suggestion workflow

In `Restricted` and `Closed` permission modes, non-team members can submit suggestions but cannot directly change the current translation:

<Steps>
  <Step title="Submit a suggestion">
    Translate a message in the web interface. If you are not a designated translator, your translation is saved as a suggestion.
  </Step>
  <Step title="Reviewer notification">
    Designated translators for the language are notified of new suggestions.
  </Step>
  <Step title="Review and accept">
    A designated translator reviews the suggestion and either accepts it (making it the current translation) or rejects it.
  </Step>
</Steps>

## Integration with upstream projects

Launchpad can share translations between a distribution package (e.g., Ubuntu's version of a package) and the upstream project it is based on. This **translation sharing** ensures that improvements made by Ubuntu translators flow back upstream, and upstream improvements are visible in Ubuntu.

The sharing relationship is configured per project series and distro series. When sharing is active, both sides see the same translation messages and suggestions.

## Translation statistics

Each template and language combination tracks:

- Total number of messages in the template.
- Number of messages with a current translation.
- Number of messages with unreviewed suggestions.
- Percentage complete.

These statistics appear on the language and template overview pages and are used to rank languages by completeness.

<Tip>
Translation statistics are computed asynchronously. If you see stale numbers after a bulk import, wait a few minutes for the stats job to run.
</Tip>
